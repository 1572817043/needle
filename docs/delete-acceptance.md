# 删除功能验收清单

> 交给 Mimo 执行，在真实 macOS 桌面 App 上逐项验收。每项记录通过/失败 + 截图证据。

## 前置条件

1. `corepack pnpm tauri:dev` 已启动，App 窗口可见
2. `~/Music/Needle` 目录存在（没有则先转换一首歌）
3. 已至少转换了 **6 首**不同的歌曲（用来验证 compact 列表和计数）

---

## 验收项

### 1. Compact 列表显示「共 N 首」

- 观察右侧「我的歌曲」面板
- **预期**：列表底部有一行灰字「共 N 首」，N = 实际歌曲数
- 截图

### 2. 删除前 5 首中的一首（Compact 可见）

1. 在右侧 compact 列表中，点击某首歌的垃圾桶图标
2. 弹出确认框，点击「确认」
3. **预期**：
   - 该行从 compact 列表消失
   - 底部「共 N 首」变为「共 N-1 首」
   - 左下角状态栏显示「已删除：XXX」
4. 截图（删除前后各一张）

### 3. 删除第 6+ 首（Compact 不可见，只看计数）

1. 先记下当前「共 N 首」的 N 值
2. 在中间面板的完整歌曲列表中找到第 6 首及之后的歌，点垃圾桶
3. 确认删除
4. **预期**：
   - 中间面板该行消失
   - 右侧 compact 面板「共 N 首」变为「共 N-1 首」
   - 即使该歌不在 compact 前 5 里，计数仍有反馈
5. 截图

### 4. 播放中删除（关键场景）

1. 点击某首歌的播放按钮，确认底部播放器正在播放
2. 在播放过程中，点击同一首歌的垃圾桶图标
3. 确认删除
4. **预期**：
   - 播放器停止、底部信息清空
   - 歌曲从列表移除
   - 状态栏显示「已删除：XXX」（不是「删除失败」）
   - 文件从 `~/Music/Needle` 中消失
5. 截图（播放中 + 删除后各一张）

### 5. 重启后不重现

1. 记下删除后「共 N 首」的数量
2. 完全退出 App（Cmd+Q）
3. 重新启动 `corepack pnpm tauri:dev`
4. **预期**：
   - 「我的歌曲」列表中的数量与退出前一致
   - 被删除的歌曲不再出现
5. 截图

### 6. 本地文件确认

1. 打开 Finder，前往 `~/Music/Needle`
2. **预期**：被删除歌曲对应的 .m4a 文件不存在
3. 截图 Finder 窗口

### 7. 终端日志检查

1. 查看启动 Tauri 的终端输出
2. **预期**：
   - 无 panic / unwrap 错误
   - 删除时有 `[delete_song]` 和 `[delete_song_from_conn]` 日志
   - 无未处理的 error 堆栈
3. 截图关键日志行
