# AGENTS.md

## 交流规则
- 始终用中文和用户交流。
- 回复保持简洁，避免无关总结。
- 不主动使用 emoji。
- 不主动创建文档文件，除非用户明确要求。

## 项目协作分工
- 主 Codex 负责：拆任务、定边界、派发给子 agent/Mimo、接收结果、复核 diff、运行验证、向用户汇报。
- `gpt-5.4` 负责：主要编码、复杂 bug 修复、跨前后端改动。
- `gpt-5.4-mini` 负责：小范围前端/测试/样式/纯函数修复。
- Mimo 负责：可独立执行的局部实现、真实界面/多模态验收、用户指定的辅助测试。
- 除非是极小修补或关键判断，主 Codex 不应独自承担大量编码。

## Agent Team 工作方式
- 优先复用固定 agent team，不为每个小任务新建一次性 agent。
- 用户负责提需求、看产品、做最终体验判断。
- 主 Codex 是最重要的总控 agent，负责理解需求、拆分任务、安排执行顺序、指挥 agent team、收口验证和汇报。
- 执行默认串行为主；只有任务互不影响、写入范围清晰、并行不会互相踩时，才安排并行。
- 建议长期角色：
  - `gpt-5.4`：主编码 agent，负责主要功能和复杂 bug。
  - `gpt-5.4-mini`：轻量修复 agent，负责样式、测试、小函数、局部前端问题。
  - Mimo：验收 agent，负责真实 App 操作、多模态观察、用户指定测试。
- 新建 agent 只在以下情况使用：任务需要隔离上下文、旧 agent 已关闭且不值得恢复、或需要并行处理互不相关的大块问题。
- 每次派发前先判断能否复用现有 team 成员；派发后由主 Codex 统一收口、验证、汇报。

## 开发约束
- 修改前先定位根因，不做随机试错。
- 涉及 bugfix 时优先补测试或可验证的最小检查。
- 修改后必须运行相关验证，再声称完成。
- 不回滚用户或其他 agent 的改动；遇到冲突先读懂再处理。
- 本项目当前以 macOS + Tauri v2 + React + TypeScript 为主。

## 当前产品边界
- 第一版只做 B 站公开视频。
- 不做登录、cookie、YouTube、歌词、批量下载、歌单同步。
- 音频正式输出为 `m4a`，转换后需要保证本地播放器可拖动、时长可靠。
